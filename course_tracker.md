## Application Specification: Sensor Dashboard

### Overview
A web application that collects data from sensors, processes it, presents results in an interactive dashboard, and generates automated reports.

### Core Features (to be refined during the course)
1. **Data Ingestion:** REST API to receive sensor readings (temperature, humidity, pressure, etc.)
2. **Data Storage:** Persistent storage of raw and aggregated sensor data
3. **Dashboard:** Real-time visualization with charts, filters, and sensor status
4. **Reports:** Automated PDF/CSV report generation on schedule
5. **Authentication:** Basic user login to protect the dashboard

### Data Model (draft — to be refined in Week 4)
- **Sensors:** id, name, type, location, status
- **Readings:** id, sensor_id, value, unit, timestamp
- **Users:** id, email, password_hash, role
- **Reports:** id, generated_at, type, file_path
